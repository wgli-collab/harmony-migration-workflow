# 交互速查卡 — AI 提示 & 你的命令

## 你的核心命令

| 命令 | 作用 | 什么时候用 |
|------|------|---------|
| `Next` | 继续下一步 | Phase 2 每个文件完成后 / 每个 Phase 过渡 / 绝大多数等待场景 |
| `fix` | 加载 07 修复环境/配置 | AI 提示 💡 时，让 AI 加载 07 修复（07 未覆盖 → AI 自行搜索） |
| `recheck` | 重新检查 / 修复 Plan | CP1 FAIL 时调整 Plan/配置后重跑，或你手动修完后让 AI 重跑当前 CP |
| `Done` | 完成 | Phase 6 Final Report 确认后 |
| `P0 ✅ P1 #8 ✅ #7 ❌ 不重要 P2 P3 📌 全部` | 审计决策 | Phase 4 审计完成后 |
| `继续` / `Continue` | 继续 | 3 轮强制暂停 / 收敛暂停 / Token 警告后 |
| `暂停` | 暂停 | 你想停下来检查时 |

## Phase 1 — PLAN

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| `✅ CP1 PASS — X/9 项通过` | Plan 没问题 | 填 06-metrics → `Next` |
| `❌ CP1 FAIL — [列失败项]` | Plan 有错 | `recheck` 让 AI 调整 Plan 后重跑 |
| `请确认分类是否合理` | 人工判断 API 分组 | 直接 `Next` 不改 / 或者说要调整哪些 |

## Phase 2 — BUILD（Loop 1：逐文件）

### 每文件完成后

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| `✅ [filename] DONE — 等待 Next | {X}/{Y} | 连续1轮通过: {streak}/5 | Token: {used}K/{budget}K` | 这个文件完了 | `Next`（进入下一个文件） |

### G2 模式检测触发

| 触发器 | AI 提示 | 你的动作 |
|--------|---------|---------|
| T1: 同违规 ≥2 连续文件 | `⚠️ [RX] 连续出现在 [FileA] 和 [FileB]` | 复制 Template A 发给 AI（补 Executor prompt）→ `Next` |
| T2: 单文件 ≥2 轮（通过） | `⚠️ [FileX] 经 N 轮通过（已记录到 metrics）` | （不暂停，无需回复） |
| —: 单文件 3 轮耗尽 | `⛔ [FileX] 3 轮未通过 — 转入 Phase 4 审计` | 单文件 3 轮耗尽 → AI 记录到 exhausted 列表，输出模板 B 分析，人工说 "Next" 继续（不在此处决策）→ 遗留问题在 Phase 4 统一决策 |
| T3: G1 streak 达 5 | `连续1轮通过: 5/5` → 自动触发 | Template C 提议 → `批量生成` 或 `Next`=接受 / `继续逐文件`=拒绝 |
| 💡 通用批量触发 | Phase 2 任何时刻（含 compaction 恢复后） | `批量生成` 或 `批量生成剩余 N 个文件` → AI 跳过逐文件 G1，一次性生成 + 逐个审计编译 + 最后汇总 |
| T4: Build 错误同依赖 | `⚠️ Build 错误指向 [dep.ets]` | Template D：先修依赖 → `Next` |
| T5: 连续 3 文件各 3 轮耗尽 | `🛑 连续 3 文件各 3 轮耗尽 — 疑似系统性异常` | 人工诊断：检查 prompt / 调整策略 / 继续 → `Next` |

### 心跳（自动，不暂停）

| AI 提示 | 含义 | 你做什么 |
|---------|------|---------|
| `📊 进度: 30/90 (33%) 首轮通过率: 28/30 (93%) 当前 Token: 120K/500K (24%)` | Phase 2 每 N 文件自动汇报（N=ceil(total/10)） | **不用管**，AI 继续跑 |
| `📊 Loop 2 Round 2/5 内循环 #1: 本轮修了 3, 剩余 2, 新发现 1` | Phase 4 每轮修复后汇报 | **不用管**，AI 继续跑 |
| `📊 Loop 3 Round 4: 当前 error=2 (上轮=5), 累计修复 8` | Phase 5 非检查点轮次汇报 | **不用管**，AI 继续跑 |

## Phase 4 — PROJECT AUDIT（Loop 2：审计-修复-重审）

### 审计报告出来后

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| `=== Phase 4 审计完成 ===` + `╔═ HARD GATE ═╗` 视觉块 | 审计完成，等你决策 | 见下方决策格式 |

### 你的决策格式

```
单条决策：
  #1 ✅          → 采纳，修
  #2 ❌ 不重要     → 驳回
  #3 ⏭️ 平台限制   → 已知 ArkTS 限制
  #4 📌 全部      → 后续提醒，Phase 6 再定

批量决策：
  P0 ✅                          → P0 全修
  P1 #8 #10 ✅  #7 #9 ❌         → 挑着修
  P2 P3 📌 全部                  → 全挂起

自然语言也行：
  "P0 P1 全修，P2 #15 ✅，其他 📌"
```

### 修复循环中

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| 增量重审发现新问题 | 修复引入了新 Critical/High | AI 追加到清单 → 回到决策界面，你继续决策 |
| `✅ CP4 PASS — 审计闭环完成` | 修复队列清空，无新问题 | 填 06-metrics → `Next`（进入 Phase 5）|
| `⚠️ Loop 2 已达上限 5 轮` | 5 轮还没清空 | 你决定：强制退出 / 继续 / 标记已知限制 |

## Phase 5 — TEST（Loop 3：测试）

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| `╔═ HARD GATE ═╗` 视觉块 + `📱 请在 DevEco Studio 模拟器上手动运行应用` | CP5 手动运行门控 | 在 DevEco 跑 → 贴结果：`通过: N 失败: N 跳过: N` |
| `⚠️ Loop 3 不收敛` | error 不降了 | 暂停 → 分析根因（test bug 还是 library bug）→ 修复 → `继续` |
| `Loop 3 已进行 N 轮` | 每 3 轮强制暂停 | `继续` / `调整策略` |
| `📊 Loop 3 Round N: 当前 error=…` | 心跳（非检查点轮次） | **不用管**，自动汇报 |
| `💡 07 §X` | 编译/环境错误 | 查 07 修复 |
| `🔍 CP5 门控自检 … 以上任一项为否 → 🛑` | CP5 自检（PASS 前自动输出） | 看到自检全部打勾 → 紧接着就是 `✅ CP5 PASS` |
| `✅ CP5 PASS` | 运行时验证通过 | 填指标 → `Next`（进入 Phase 6）|

## Token Checkpoint（所有 Phase 边界）

| AI 提示 | 含义 | 你回什么 |
|---------|------|---------|
| `📊 TOKEN: 累计=XXXK / 预算=YYYK (ZZ%) \| compaction=N \| ≥80%⚠️ / ≥95%🛑` | 正常 | `Next` |
| `⚠️ 消耗已达 80%` | 接近上限 | `继续` / `暂停` |
| `🚫 消耗已达 95%` | 硬上限 | 收尾当前 Phase → 停止新工作 |
| `上下文已压缩` | 视为 ≥50% | 注意消耗 |
